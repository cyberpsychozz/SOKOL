# Квантование модели

В этой папке лежит небольшой пайплайн для перевода `best.pt` в ONNX и
последующего квантования ONNX-модели в INT8.

## Входные и выходные файлы

- Входные веса по умолчанию: `../best.pt`.
- Экспортированная FP32-модель: `output/best_fp32.onnx`.
- Dynamic INT8-модель: `output/best_int8_dynamic.onnx`.
- Static INT8-модель: `output/best_int8_static.onnx`.

Скрипт инференса в репозитории уже умеет принимать ONNX-модели, поэтому
полученный файл можно передать так:

```bash
python MVP/src/orangepi_inference.py --model quantization/output/best_int8_dynamic.onnx
```

## Установка зависимостей

```bash
pip install -r requirements.txt
```

## Dynamic INT8-квантование

Dynamic-квантование не требует изображений. Оно квантует веса модели и является
самым быстрым способом получить более компактный INT8 ONNX-файл.

```bash
python quantization/quantize_best.py --weights best.pt --mode dynamic
```

Флаг `--simplify` добавляйте только в том случае, если ваша локальная среда
поддерживает упрощение ONNX-графа.

## Static INT8-квантование

Static-квантование использует репрезентативные изображения для калибровки не
только весов, но и активаций модели. Для детекторов это обычно более правильный
INT8-вариант, потому что калибровочные изображения помогают подобрать реальные
диапазоны значений внутри сети.

Используйте 50-200 настоящих кадров, похожих на те, которые модель будет видеть
при работе на устройстве:

```bash
python quantization/quantize_best.py ^
  --weights best.pt ^
  --mode static ^
  --calibration-images datasets/sokol/valid/images ^
  --calibration-limit 100
```

После квантования сравните FP32 и INT8-модели на одном и том же видео или наборе
валидационных изображений. INT8 стоит принимать только если выигрыш в скорости
или размере оправдывает возможную потерю точности.
